Role of Superadmin

  • Can create and view admin and smartxengineer
  • Create and view organization unit
  • Can assign a particular device to admin
  • View all the devices
    • Steps for creating Admin and Smartxengineer
    After logging in to the superadmin account have to navigate to users page where they can click add users button to create a admin. On clicking that following fields have to be entered. 1. Email-Id: this field contains the email address of the admin or smartxengineer which we are creating. This field is used as userid while login time and assigning the users to particular device. 2. Password: this field is used while logging in and its hidden. 3. User name: this field contains the name of the user and it’s a required field. 4. Gender: this field is a dropdown which shows male female other 5. Age: this field is the age of the user and a required field. 6. Address: this field contains the Address of the user and a required field 7. User type: this field is a drop down which decides the type of the user.we can select admin or smartxengineer from the dropdown. 8. If the usertype is admin another field will appear as extra role click that checkbox so that your admin will function also as service provider. After entering all the fields on clicking register button our registered user is saved and that can be viewed in view page.
    Add Orgnization unit
    Superadmin can add organizational unit other users can view. In this page there will be add oranisation unit button on clicking that following fields can be seen. 1. Organization Id: it should contain the unique Id in string type and a required field. 2. Organization Name: It should contain the name of that Organization. 3. Service state: It should contain the state. 4. Address1: It should contain the address line1. 5. Address2: It should contain the address line2. 6. District: this field contains the district name 7. State: this field contains the state name 8. Pincode: this field contains the pincode of the place 9. Logo upload: here we can upload the logo of your organization in jpeg,png,jpg formats. After entering all the fields on clicking register button our organisation unit is saved and that can be viewed in view page.
    Assign Device
    This page is accessible only to superadmin. Here he can assign a particular device to admin. If logged in as superadmin in this page there will be add button on clicking that following fields can be seen. 1. Device id: this field is a dropdown which shows device id of all the created devices. if a device is already assigned to admin then it will not be shown in dropdown. 2. Admin: this field is a dropdown which shows user id of all the usres whose usertype is admin. After entering all the fields on clicking register button our registered device and admin is saved and that can be viewed in view page.

    Role of Admin

  • Can create and view Service Provider
  • Can allocate a device to service provider
    • Steps for creating service provider
    ● The user have to login to the account using the credentials gave by developers. ● After logging in the user should navigate to the service provider registration page. In this page there will be add button on clicking that enter the following fields in order to create a service provider. 1. email: this field contains the mail of the serviceprovider. This field is used as userid while login time and can it is viewed as userid in view page also. 2. Password: this field is used while logging in and its hidden. 3. User name: this field contains the name of the user and it’s a required field. 4. Gender: this field is a dropdown which shows male female other 5. Age: this field is the age of the user and a required field. 6. Address: this field contains the Address of the user and a required field After entering all the fields on clicking register button our registered user is saved and that can be viewed in view page.
    Allocate Device to Service Provider
    This page is accessible to admin where the admin can map device with service provider .In this page there will be add button on clicking that following fields can be seen. 1. Device id: this field is a dropdown which shows the device id of the devices which is assigned to that particular admin. 2. Service provider: this field is a dropdown which shows the user id whose usertypes are service provider. the admin can select the service provider accordingly. After entering all the fields on clicking register button our registered mapping is saved and that can be viewed in view page.

    SmartX Engineer

    Role of SmartX Engineer
  • Can create and view Device
    • Steps for creating Device
    After logging in with the smartxengineer credentials the user have to navigate to devices page. There we have to click add device button and should enter the following fields. 1. Deviceid: The device id must be exactly 12 digits in string and it’s a required field. 2. Devicetype: This field contains a dropdown with two options wearable and handheld.this can be selected based on the device type. It’s a required field 3. Device manufacturer: This field contain the device manufacturer name in string type and it’s a required field. 4. Imei: This field contains the imei number of the device and a required field. 5. Device test engineer:This field contains the name of the person who tested the device and a required field. 6. Date of release: This field contains the date on which the device is released in DD/MM/YYYY format. 7. Sim number: This field contains the sim number of the device and it’s a required field. After entering all the fields on clicking register button our device is saved and that device can be viewed in view page.
